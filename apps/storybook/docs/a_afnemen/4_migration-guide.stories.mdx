## Versie v2.0.0

In deze toekomstige versie willen we enkele verbeteringen doorvoeren doorheen de componenten.
- onder meer is het de bedoeling om de huidige "elements" te laten vervallen.
- daarnaast willen we ook enkele componenten vervangen door CSS klasses.
- ook form gerelateerde componenten zullen vervangen worden door `next` componenten.

❗De componenten worden in v2.0.0 vervangen door de huidige `next` versie.

❗De huidige next componenten verliezen na elke major release dan ook de `-next`-suffix uit hun tag.

### Elements vervangen

We willen alle huidige [elements](/docs/afnemen-artifacts--documentatie#domg-wcelements) (`@domg/wc-elements`) laten vervallen. Dit heeft enkele redenen:
- "customized built-in elements" (custom elements met `is=""`, bv.: `<button is="vl-button">`) is een feature die nooit ondersteund zal worden door Apple & Safari (zie [caniuse](https://caniuse.com/custom-elementsv1), [MDN](https://developer.mozilla.org/en-US/docs/Web/API/CustomElementRegistry/define#browser_compatibility) en [bugs.webkit.com](https://bugs.webkit.org/show_bug.cgi?id=182671))
- het wordt ook niet ondersteund door Lit
- voor heel wat elementen is het niet nodig om een eigen component te hebben, CSS klasses zijn voldoende
- daarnaast maakt dit het ons ook mogelijk om consistentere codebase te verkrijgen, o.a.:
    - `data-vl`-prefix overal wegwerken (die enkel nodig was voor de `is=""`-feature)
    - consistentere API (alle componenten kunnen extenden van `LitElement`)
    - wegwerken van BaseElementOfType / `dress` lifecycle door `Lit`'s lifecycle te gebruiken

### Form herwerking
> Meer informatie over het gebruik van de nieuwe form componenten kan je terugvinden in de [form demo](/docs/ontwerp-form-demo--documentatie).

Vanaf v2.0.0 zullen de huidige form gerelateerde componenten niet meer ondersteund worden.

De huidige form gerelateerde componenten zullen vervangen worden door `next` componenten, reeds beschikbaar onder de [form package](/docs/afnemen-artifacts--documentatie#domg-wcform) (`@domg-wc/form`).
Het heeft de voorkeur om daar waar mogelijk, zeker in nieuwe toepassingen, alvast de `next` componenten te gebruiken.
