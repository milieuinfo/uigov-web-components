## Overzicht

Alle web-componenten van Departement Omgeving zijn beschikbaar via npm artifacts onder de npm scope `@domg-wc`;
__common-utilities__ bevat technische code, de overige 4 artifacts: __elements__, __components__, __map__ en
__sections__ bevatten de web-componenten. Alle web-componenten zijn ook beschikbaar via de CDN, zie
['fat-js: domg-wc'](#fat-js-domg-wc).

> ** ⚠ Waarschuwing**: er is een elements / components concept gedefinieerd, dit is een streefdoel, de huidige code
volgt dit concept in grote lijnen maar niet volledig. Afwijkingen komen voor!


## @domg-wc/common-utilities

__common-utilities__ is een technische bibliotheek met basis code en utilities.


## @domg-wc/elements

Een __element__ is geïmplementeerd als een native html-element en heeft technisch volgende eigenschappen:

- extends steeds van een sub-klasse van HTMLElement (bvb. HTMLButtonElement)
- dus steeds te gebruiken met 'is='; bvb. `<button is="vl-button"></button>`
- heeft __geen__ shadow DOM
- styling wordt globaal gedefinieerd, dit gebeurd eenmalig en automatisch
- de bulk van de styling komt van DV (Digitaal Vlaanderen)


## @domg-wc/components

Historisch werd een __component__ geïmplementeerd als een `BaseElement`: een eigen basis klasse beschikbaar in
@domg-wc/common-utilities via de methode `BaseElementOfType`. Nieuwere componenten worden geïmplementeerd m.b.v.
[LitElement](https://lit.dev/docs/api/LitElement/); het doel op (lange) termijn is van alles (ook van de elementen,
sections en map) LitElementen te maken.

Een __component__ heeft technisch volgende eigenschappen:

- extends van BaseElement of van LitElement
- dus steeds te gebruiken als custom tag; bvb. `<vl-breadcrumb></vl-breadcrumb>`
- heeft __steeds__ een shadow DOM
- de styling komt mee met de component (wordt geïnjecteerd in de shadow DOM)
- element styling wordt waar nodig voorzien in de shadow DOM


## @domg-wc/sections

Onder __sections__ zitten web-componenten specifiek voor Departement Omgeving. Het zijn geen bouwstenen
zoals de andere __componenten__ of __elementen__. Het __sections__ artifact bevat: __vl-header__, __vl-footer__,
__vl-accessibility__, __vl-cookie-consent__, __vl-cookie-consent-opt-in__, __vl-cookie-statement__ en __vl-privacy__.


## @domg-wc/map

De __map__ is de grootste en uitgebreidste component. Het is een kaart component die achterliggend
[OpenLayers v6.14.1](https://openlayers.org/en/v6.14.1/doc/) gebruikt. De functionaliteit zit in sub-componenten
die een plugin opzet volgen.


## fat-js: domg-wc

Specifiek voor legacy toepassingen is er een 'fat-js' variant beschikbaar. Via de CDN van Departement Omgeving kan
je daar geversioneerd naar refereren conform: `https://cdn.omgeving.vlaanderen.be/domg/wc/domg-wc-1.18.0.min.js`. Bij
de import worden alle web-componenten (elements, components, sections en map) geregistreerd. Zie
[CDN Bundel - fat-js](/story/afnemen-cookbook--page#cdn-bundel---fat-js) onder `Cookbook` voor meer informatie.
