## Doel

Met de UIG web componenten en deze bijhorende documentatie streven we er naar meer te zijn dan een verzameling
componenten. Het doel is een design system aan te bieden.


## Wat is een Design System

Een uitgebreide uitleg over wat een design system is vind je in deze 2 artikels:
[The Design System Ecosystem](https://bradfrost.com/blog/post/the-design-system-ecosystem/) en
[Design Systems are for user interfaces](https://bradfrost.com/blog/post/design-systems-are-for-user-interfaces/).

Een **design system** is een **structureel geheel** dat aangeboden wordt om web-toepassingen te bouwen met een
**uniforme** en **gebruiksvriendelijke interface**. Het is een systeem dat uit sub-systemen bestaat, in ons geval:

- de verschillende componenten - zitten in de [artifacts](/docs/afnemen-artifacts--documentatie) die we aanbieden
- een font - voorzien door DV, wordt in de componenten gebruikt
- iconen - zitten in het [Icon Font](/docs/beheren-icon-font--documentatie) van DV; afneembaar via
  [`<vl-icon-next>`](/docs/components-next-icon--documentatie) en [`<vl-icon>`](/docs/elements-icon-icon--documentatie)
- een kleuren palet - wordt in de componenten gebruikt, wordt nog niet expliciet aangeboden
- een layout / grid opzet - wordt via de elementen aangeboden, dit zal wijzigen in de 'next'-opzet
- documentatie van het geheel - deze Storybook documentatie
- een set van leidraden - de pagina's te vinden ander RICHTLIJNEN, RECEPTEN en ONTWERP
- goede voorbeelden - de stories en code snippets (via de 'Show code'-knoppen)


## Wat doet UIG (niet)

De focus van het UIG-team ligt op het **uitwerken** en **ondersteunen** van dit **design system**. Het doel is dat
ontwikkelteams m.b.v. de aangeboden bouwblokken frontend toepassingen kunnen bouwen. Het feitelijk bouwen van
toepassingen is de verantwoordelijkheid van de ontwikkelteams. UIG kan begeleiden en zal ondersteunen waar nodig.
