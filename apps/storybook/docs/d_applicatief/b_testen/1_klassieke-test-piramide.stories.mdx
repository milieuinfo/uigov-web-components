# Klassieke Test Piramide

<br/>
<img src={klassiekeTestPiramide} alt="Klassieke Test Piramide" height="350px"/>
<br/>
<br/>

De meest simpele variant van de test piramide gaat op voor alle type toepassingen. Het is het minimale wat dient te
gebeuren rond testen, de feitelijke uitvoering ligt open. Volgende concepten zitten vervat in de piramide:

- het doel is minimale overlap te hebben tussen de niveaus: wat al getest is op een lager niveau dient niet meer getest
  te worden op een hoger niveau
- naarmate je dichter bij de top komt
  * stijgt de complexiteit van de testen
  * stijgt de implementatie tijd
  * stijgt de doorlooptijd van uitvoering
  * daalt de hoeveelheid van 'dat' type testen

> **conceptueel: probeer 'wat' je wil testen zo laag als mogelijk in de piramide te doen**
