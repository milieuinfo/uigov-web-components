## Concept

In de next / v2 opzet worden de [elementen weggewerkt](/docs/planning-aanpak-v2-2024-v3-2025--documentatie#element-complicatie),
dit heeft impact op de globale styling. De elementen uit v1, bvb. button moeten gebruikt worden als `<button is="vl-button">`;
hierdoor hebben die geen shadow-dom en wordt die css globaal (op het document) voorzien. In v2 wordt `button` een
volwaardige web-component, te gebruiken via `<vl-button>`, met een shadow-dom. Hierdoor kan de 'button' style
nu in de shadow-dom gestoken worden.

Alle styling voor componenten wordt in v2 op die manier aangeboden. Naast het stylen van componenten is er nog andere
opmaak nodig:

- **font definities** om overal het Vlaanderen font te gebruiken
- **mixins** en **variabelen** om duplicatie tegen te gaan
- **layout** en **witruimte** voor pagina-opmaak
- opmaak voor **native** html elementen

In dit document wordt de desbetreffend opmaak beschreven: alles wat **niet** in de shadow-dom van een component kan.


## Conventies

- alle css wordt geschreven als css-in-ts en aangeboden als **css-in-js**
- de css volgt de [**BEM**](https://getbem.com/) methodologie
- om styling eenduidig te specifiëren worden [**css variabelen**](https://developer.mozilla.org/en-US/docs/Web/CSS/Using_CSS_custom_properties) gebruikt
- css variabelen worden globaal gedefiniëerd maar kunnen lokaal (in een component) een afwijkende waarde krijgen
- we gebruiken het **mixin concept** in de vorm van geparametriseerde typescript methodes

### Container

alles wat orde en structuur geeft. Dit zijn containers voor ander componenten. Om deze te implementeren maakten we
een aantal keuzes:

- geen shadow-dom: dit vermijdt de complexiteit van shadow-dom's in shadow-dom's

[CSS Grid Layout](https://css-tricks.com/snippets/css/complete-guide-grid/) die native ondersteund wordt in een browser
Op die manier kan de normale documentatie die daarover bestaat geraadpleegd worden en voelt het vertrouwd voor
ontwikkelaars die er ervaring mee hebben.

grid / group / margin / padding
