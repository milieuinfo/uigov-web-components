# Project Structuur

## Beschrijving

Hieronder staat de projectstructuur van de [uigov-web-components](https://github.com/milieuinfo/uigov-web-components)
repo beschreven, achter de pijltjes staan de artifacts gespecifieerd die uit de sub-projecten komen.

Er zitten 2 applicaties in de repo: **Storybook** en **Playground**, waarbij storybook streeft naar het aanbieden van
zo correct mogelijke informatie en documentie. Playground heeft momenteel eerder een experimentele insteek.

De feitelijke componenten zitten opgesplitst in een aantal bibliotheken onder de libs folder. Ze worden aangeboden
als npm packages die gepubliceerd worden onder de scope **@domg-wc**.

## Structuur

```
─ uigov-web-components/ [NX]
  ├─ apps/
  │  ├─ playground/
  │  ├─ playground-e2e/ (Cypress testen tov playground)
  │  ├─ storybook/
  │  ├─ storybook-e2e/ (Cypress testen tov storybook)
  │   -> worden niet als artefact gepubliceerd
  ├─ libs/
  │  ├─ common/
  │  │  ├─ utilities/
  │  │  │   -> @domg-wc/common-utilities
  │  │  ├─ storybook/
  │  │  │   -> @domg-wc/common-storybook
  │  ├─ components/
  │  │   -> @domg-wc/components (http-error-message / proza-message)
  │  ├─ elements/
  │  │   -> @domg-wc/elements
  │  ├─ map/
  │  │   -> @domg-wc/map
  │  ├─ sections (accessibility / cookie / privacy / ...)
  │  │   -> @domg-wc/sections
  │  │   -> @domg-wc/fat-sections [TODO]
  │  ├─ support/
  │  │   -> @domg-wc/react-support [TODO]
  │  │   -> @domg-wc/test-support [js]
  ├─ resources/
```
